# Cloud-Security-AI-Threat-Detection-Simulation-Experiment
Experimental setup, approach, and full written report of AWS and Azure cloud environments simulating penetration testing and misconfiguration detection using native AI threat detection tools integrated with DevSecOps principles. AWS utilization of machine learning native AWS based GuardDuty and Security Hub center. Microsoft Defender used for Azure. Penetration testing performed using Nmap on live cloud servers for both providers. Snyk code vulnerability scanning reports illustrate how to remediate and patch code repositories before they are launched into a cloud environment. Part of the final project for CYBV 579-Cloud Security as part of the Master of Science in Cyber and Information Operations degree program at the University of Arizona College of Information Science, an NSA designated CAE-CO (Cyber Operations), Cyber Defense (CAE-CD), and Research (CAE-R) university. The paper title is "Enhancing Cloud Security through Machine Learning Driven Threat Detection and DevSecOps Integration".
